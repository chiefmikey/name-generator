# Retro: Dev Server + S3/CloudFront Deployment (2026-03-10)

## What Worked
- **Parallel execution**: S3 bucket + ACM cert creation ran in parallel with webpack-dev-server + favicon work. Total wall time ~5 minutes for all infra.
- **ACM DNS validation**: Certificate issued in under 30 seconds with Route53 DNS validation.
- **S3 website hosting with CloudFront**: Clean pattern — S3 as custom origin (http-only), CloudFront handles HTTPS + caching + SPA fallback.
- **Reusing existing hullowurld.com hosted zone**: No new domain purchase needed. Just a subdomain record.

## What Didn't Work
- **CloudFront deployment time**: 4.5 minutes to deploy. No way around this — it's propagating to edge locations globally. Just have to wait.
- **Shell variable collision**: zsh has `status` as a read-only variable. Had to use `cf_status` instead in the polling loop.

## What Was Surprising
- **PriceClass_100 is sufficient**: Only serves from North America and Europe edge locations but costs less. Perfect for a fun side project.
- **The existing hullowurld.com setup was already a static S3+CloudFront site** (pacmane-game). Same pattern, easy to replicate.

## Deployment Process
```bash

aws s3 sync public/ s3://name-hullowurld-com/ --delete --cache-control "max-age=86400"
aws cloudfront create-invalidation --distribution-id E1IMS0MNM7TTIU --paths "/*"
```
